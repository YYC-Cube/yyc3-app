# 文档标准化指南

本文档详细说明邮件系统文档中心的标准化规范和维护流程，帮助团队成员创建和维护高质量、一致性的文档。

## 📋 标准规范

### 1. 文档头部标准化

所有文档必须包含标准的JSDoc风格头注释：

```markdown
/** 
 * @file 文档文件名 
 * @description 文档简要描述 
 * @module 所属模块 
 * @author 作者信息 
 * @version 文档版本 
 * @created 创建日期 
 * @updated 更新日期 
 */
```

* `@file`: 文档的文件名
* `@description`: 文档的简要描述，说明文档的主要内容和目的
* `@module`: 文档所属的模块或分类
* `@author`: 文档的创建者
* `@version`: 文档版本号，遵循语义化版本规范
* `@created`: 文档创建日期，格式为YYYY-MM-DD
* `@updated`: 文档最后更新日期，格式为YYYY-MM-DD

### 2. 文件命名规范

* 统一使用连字符(`-`)分隔单词，避免使用下划线或驼峰命名
* 文件名应简洁明了，反映文档内容
* 目录名同样使用连字符分隔
* README.md用于目录说明文档

### 3. 更新时间统一

* 在文档头部的`@updated`标签中指定最后更新日期
* 当文档内容发生变化时，必须更新此日期

### 4. 文档版本控制

* 遵循语义化版本规范(MAJOR.MINOR.PATCH)
* 版本号应与相关代码版本保持同步
* 重大更新增加主版本号，功能更新增加次版本号，错误修复增加补丁版本号

### 5. 文档元数据

* 为每个文档添加相关标签和关键词
* 使用`@module`标签指定文档分类
* 确保文档内容与元数据保持一致

## 🛠️ 标准化工具

我们提供了以下工具来帮助自动化文档标准化过程：

### 1. 文档头部标准化工具

**用途**：自动为文档添加标准JSDoc风格的头部注释

**使用方法**：

```bash
# 为单个文件添加标准头部
./scripts/normalize-doc-headers.sh -f 文件路径 "文档描述"

# 批量处理目录中的所有Markdown文件
./scripts/normalize-doc-headers.sh -d 目录路径 "默认文档描述"

# 处理整个wiki目录
./scripts/normalize-doc-headers.sh -d . "邮件系统文档"
```

**示例**：

```bash
# 为Home.md添加标准头部
./scripts/normalize-doc-headers.sh -f ./Home.md "邮件系统整体架构和功能概述"

# 批量处理Services目录
./scripts/normalize-doc-headers.sh -d ./Services "服务配置和API文档"
```

### 2. 搜索索引生成工具

**用途**：自动扫描文档并生成搜索索引，更新搜索页面

**使用方法**：

```bash
# 使用默认设置生成索引
./scripts/generate-search-index.js

# 指定wiki目录
./scripts/generate-search-index.js --wiki-dir ./

# 仅输出索引文件，不更新HTML
./scripts/generate-search-index.js --output index.json --no-update-html
```

**参数说明**：

* `--wiki-dir`：指定wiki文档根目录(默认：当前目录)
* `--output`/`-o`：指定索引输出文件路径
* `--no-update-html`：不更新search.html文件中的索引
* `--search-html`：指定search.html文件路径
* `--debug`：启用调试日志
* `--help`/`-h`：显示帮助信息

## 🔍 文档搜索功能

我们提供了文档搜索页面，方便快速查找文档：

**搜索页面**：`search.html`

**功能特性**：

* 在文档标题、描述、标签和关键词中搜索
* 搜索结果高亮显示匹配文本
* 支持大小写不敏感搜索
* 点击搜索结果直接跳转到对应文档

**使用方法**：

1. 打开`search.html`页面
2. 在搜索框中输入关键词
3. 点击搜索按钮或按Enter键开始搜索
4. 浏览搜索结果，点击标题跳转到文档

**搜索索引更新**：
当添加或修改文档后，使用搜索索引生成工具更新搜索索引：

```bash
./scripts/generate-search-index.js
```

## 📋 文档审核流程

为确保文档质量和一致性，我们建立了文档审核机制：

### 1. 审核清单

文档审核时应检查以下项目：

* [ ] 文档头部是否符合标准
* [ ] 文件名是否遵循命名规范
* [ ] 更新时间是否为最新
* [ ] 版本号是否适当
* [ ] 是否添加了相关标签和关键词
* [ ] 文档内容是否准确、清晰、完整
* [ ] 格式是否一致
* [ ] 链接是否有效

### 2. 审核责任

* 文档作者负责确保文档符合基本标准
* 团队负责人或技术负责人进行最终审核
* 所有审核结果应记录并反馈给文档作者

### 3. 审核频率

* 新文档必须经过审核才能发布
* 现有文档每季度进行一次复审
* 代码重大更新后相关文档应同步审核

## 📊 文档维护计划

### 1. 定期维护

* 每周：检查并修复失效链接
* 每月：更新搜索索引，确保所有文档可被搜索
* 每季：进行文档全面审核和版本更新

### 2. 维护工具

* 使用文档头部标准化工具确保头部一致性
* 使用搜索索引生成工具保持搜索功能正常
* 使用文档模板创建新文档

### 3. 贡献指南

1. 使用标准模板创建新文档
2. 确保文档符合所有标准规范
3. 提交前运行头部标准化工具
4. 更新相关索引
5. 请求审核

## 📝 最佳实践

1. **保持简洁明了**：使用简洁的语言和清晰的结构
2. **使用Markdown**：所有文档使用Markdown格式
3. **添加示例**：尽可能提供代码示例和使用案例
4. **定期更新**：确保文档与实际实现保持同步
5. **使用图表**：适当使用图表说明复杂概念
6. **链接相关文档**：引用相关文档以便用户深入了解
