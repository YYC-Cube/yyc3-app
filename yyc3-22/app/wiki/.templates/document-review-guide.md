# 文档审核指南

## 审核目标

确保所有wiki文档符合项目标准化要求，提高文档质量和一致性，提升用户体验。

## 审核清单

### 1. 文档头部规范
- [ ] 包含标准文档头注释（@file, @description, @module, @author, @version, @created, @updated）
- [ ] 头注释格式正确，无语法错误
- [ ] 所有必填字段均已填写

### 2. 文件命名规范
- [ ] 文件使用连字符(-)分隔单词（如：file-name.md）
- [ ] 文件名描述性强，能反映文档内容
- [ ] 避免使用特殊字符和空格

### 3. 内容质量
- [ ] 内容完整，覆盖所需信息
- [ ] 无语法和拼写错误
- [ ] 代码示例正确可用
- [ ] 步骤清晰，易于理解和执行

### 4. 格式规范
- [ ] 使用Markdown格式正确
- [ ] 标题层级合理（#、##、###等）
- [ ] 代码块使用正确的语法高亮
- [ ] 表格、列表等格式化元素正确使用

### 5. 元数据完整
- [ ] 包含@tags和@keywords，便于检索
- [ ] 更新时间信息正确且格式统一
- [ ] 版本号与代码版本保持同步

## 审核流程

1. **提交审核**：作者完成文档后，提交给审核员
2. **初步审核**：审核员检查文档是否符合基本规范
3. **详细审核**：审核员按照审核清单进行详细检查
4. **反馈修改**：审核员提供修改建议
5. **最终确认**：作者修改后，审核员进行最终确认
6. **发布**：文档通过审核后正式发布

## 审核责任

- **文档作者**：确保文档符合基本规范，内容准确完整
- **审核员**：严格按照审核清单进行检查，提供建设性反馈
- **文档管理员**：定期抽查文档质量，更新审核标准

## 审核频率

- 新文档：提交时必须经过审核
- 现有文档：每季度进行一次例行审核
- 重大更新：更新后必须重新审核

## 质量指标

- 文档规范符合率：目标≥95%
- 审核通过率：目标≥90%
- 文档用户满意度：目标≥85%

## 工具支持

- 使用文档模板（.templates/doc-template.md）确保格式一致
- 考虑引入自动化检查工具，如markdownlint
