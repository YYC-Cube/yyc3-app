# 📜 CHANGELOG

记录 RediOps API 项目的结构演进与交付历史。

## [v1.0.0] - 2025-10-28

### 🎯 核心功能

- ✅ `/status` 服务健康检查接口
- ✅ `/user/register` 用户注册接口
- ✅ `/user/login` 用户登录接口
- ✅ `/user/info` 获取当前用户信息
- ✅ `/user/update` 修改用户信息
- ✅ `/user/logout` 注销登录（前端清除 token）

### 🧱 数据库结构

- ✅ 创建数据库 `YYC3_API`
- ✅ 创建表 `api_users`，字段含注释、索引、默认值
- ✅ 用户模型封装：`findByEmail`, `create`, `findById`, `updateById`

### 🔐 安全模块

- ✅ JWT 鉴权中间件 `auth.js`
- ✅ `.env.example` 配置项标准化

### 📦 项目结构

- ✅ 模块化目录：routes, controllers, models, middleware, validators
- ✅ 所有接口保持响应结构统一，含 `code`, `message`, `data`

## 📦 错误码列表

| 错误码 | 描述               | 建议处理方式         |
| ------ | ------------------ | -------------------- |
| 1001   | 参数缺失或格式错误 | 检查请求体结构       |
| 1002   | 未提供 token       | 引导用户重新登录     |
| 1003   | token 无效或过期   | 清除 token，重新登录 |
| 1004   | 用户不存在         | 检查用户 ID 或邮箱   |
| 1005   | 服务异常           | 记录日志，联系管理员 |
| 1006   | 权限不足           | 限制访问，提示身份   |
| 1007   | 请求过于频繁       | 限流提示             |
| 1008   | 登录频率过高       | 限制登录尝试         |
